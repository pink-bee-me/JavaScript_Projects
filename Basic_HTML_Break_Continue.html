Javascript Flow / Loop Control Statements

Definitions and Explanations:

Break Statement:
These are used to terminate a loop immediately, even if the condition is not yet fulfilled.It is used to exit a loop early, breaking out of the enclosing curly braces.Break statements are also used with switch statements. Adding a label is optional. The key take away is that a break statement quits executing the loop- jumping out of the loop to execute the next code that comes after the loop.

Syntax for Break Statement:
break [label];


Continue Statement:
The continue statement tells the interpreter to immediately start the next iteration of the loop and skip the remaining code block of the current iteration in process. When a continue statement is encountered, the program flow moves to the loop check expression immediately and if the condition remains true, then it starts the next iteration, otherwise the control comes out of the loop.

Syntax for Condition Statements:
condition [label];


Label Statements:
These can be used with both break and continue statements. Their purpose is the offer a more precise means to control the flow within a program. A label is simply an identifier that is followed by a colon. The Label statement comes just before a statement or a block of code.

Syntax for Label Statements:
labelName:
statement

NOTE:
Line breaks are NOT allowed between the 'continue' or 'break' statement and its label name. Also, there should not be any other statement in between a label name and associated loop.
